# 🔥 OnFire App — Prevent Sleep Tray Utility

---

# 🧠 Project Overview

This project is a **Windows desktop application** built using:

* Electron
* TypeScript
* Node.js

The goal is to create a **lightweight system tray utility** that prevents the computer from entering sleep mode or turning off the display.

Additionally, the app will include:

🖱 **Mouse Idle Protection**

If the mouse remains idle for a configurable number of minutes, the cursor will move slightly to simulate activity.

---

# 🎯 Core Modes

## 🔥 On Fire Mode

Main system prevention mode.

When **enabled**:

* Prevent system sleep
* Prevent display sleep
* Move mouse slightly when idle
* Update tray icon to active state
* Persist state

When **disabled**:

* Restore system defaults
* Stop idle mouse movement
* Update tray icon to inactive state


---

## Mouse Idle Logic

Implementation:

```ts
import robot from "robotjs";

let intervalId: NodeJS.Timeout | null = null;

let lastMoveTime = Date.now();

export function startMouseJiggle(
  idleMinutes = 5,
  movePixels = 2
) {

  const idleMs = idleMinutes * 60 * 1000;

  intervalId = setInterval(() => {

    const pos = robot.getMousePos();

    const now = Date.now();

    const idleTime = now - lastMoveTime;

    if (idleTime >= idleMs) {

      const originalX = pos.x;
      const originalY = pos.y;

      robot.moveMouse(
        originalX + movePixels,
        originalY + movePixels
      );

      setTimeout(() => {

        robot.moveMouse(
          originalX,
          originalY
        );

      }, 300);

      lastMoveTime = Date.now();

    }

  }, 10000);

}

export function stopMouseJiggle() {

  if (intervalId) {

    clearInterval(intervalId);

    intervalId = null;

  }

}

export function updateMouseActivity() {

  lastMoveTime = Date.now();

}
```

# 🪄 Tailwind CSS Integration

Added Tailwind CSS as a dev-time build tool.

Version: `^3.4.0`

Config file:

```text
tailwind.config.js
```

Source CSS (Tailwind input):

```text
src/renderer/input.css   ← @tailwind directives + all custom CSS
```

Output CSS (used by renderer):

```text
src/renderer/styles.css  ← generated output OR hand-crafted fallback
```

Build commands:

```bash
npm run build:css     # compiles Tailwind → styles.css
npm run build         # build:css + tsc
npm run dev           # build:css + tsc + electron .
```

Note:

* `styles.css` is hand-crafted (works immediately, no build step needed)
* `input.css` is the Tailwind source; running `npm run build:css` regenerates `styles.css`
* HTML uses semantic component class names (not Tailwind utility classes in markup)
